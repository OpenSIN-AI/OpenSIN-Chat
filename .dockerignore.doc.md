# `.dockerignore` CoDocs

This file tells Docker which files and directories to exclude from the build context when building the OpenSIN-Chat production image via `docker/docker-compose.yml`.

## What gets excluded

- **Git metadata** — `.git`, `.gitignore` are not needed inside the image.
- **Node dependencies** — `node_modules` in the root, `frontend/`, `server/`, and `collector/` are reinstalled inside the image by the Dockerfile, so shipping the host copies only bloats the context.
- **Build outputs** — `frontend/dist`, coverage reports, and log files are regenerated during the build.
- **Runtime data** — `server/storage`, `collector/hotdir`, and `collector/outputs` are mounted as volumes at runtime and must never be baked into the image.
- **Secrets / env files** — `docker/.env`, `.env`, and `.env.example` are mounted at runtime or contain example credentials that should not be copied into the image.
- **Development noise** — `.DS_Store`, `.vscode`, `.idea`, swap files, and test artifacts.
- **Chromium download artifacts** — `chrome-linux/` and `chrome-linux.zip` are downloaded inside the image on ARM builds.
- **Documentation** — all `*.md` files are excluded, except `README.md` and the `docs/**/*.md` tree (required by the frontend `sync:docs` prebuild hook so the in-app documentation under `/docs` stays in sync with the repo source).

## Why this matters

Without this file, the Docker build context exceeded 2 GB and repeatedly caused OrbStack's Docker daemon to crash with `failed to receive status: rpc error ... EOF`. After adding `.dockerignore`, the context shrinks to a few hundred megabytes and the build can complete the compile/install steps. The final image export step may still depend on OrbStack's available disk space.

## Related files

- `docker/Dockerfile` — multi-stage build that reinstalls dependencies and rebuilds the frontend
- `docker/docker-compose.yml` — mounts `docker/.env`, `server/storage`, and collector directories
- `server/.env.example` — example configuration; must not be baked into the image
